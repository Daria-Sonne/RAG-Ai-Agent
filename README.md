# Analyzing-Agent-with-local-llm
## 1.Цель
- Разработать систему, которая:
 - Принимает на вход текстовый вопрос на естественном языке
 - Выдаёт текстовый аналитический ответ, опираясь только на данные из предоставленного CSV-датасета и LLM, доступную локально.

## 2. Данные
- Дан CSV-файл hotel_tickets_data.csv с бизнес-данными.
- Подробное описание полей (типы, смысл колонок)  дано отдельно в документе ТБ-Отели и а_б.pdf

## 3. Типы вопросов
### Примеры:
 - «Как менялась цена авиабилетов в 2023 году?»
 - «Какая программа лояльности наиболее активно покупает авиабилеты?»
### Могут быть и более сложные вопросы, которые потребуют от агента комплексного анализа:
 - «Каких клиентов можно считать серыми реселлерами?» (здесь агенту понадобится не только понять что тут имеется в виду под «серыми реселлерами», но и провести всесторонний анализ множественных показателей, чтобы однозначно установить портрет мошенников)
 - Где мы “дарим” деньги: недооценённые высокозагруженные даты/направления (здесь агенту нужно понять, какие направления недооценены по загруженности/стоимости)
### Ответы должны быть аналитическими, то есть:
 - содержать числа (конкретные значения, проценты, доли и т.п.),
 - сопровождаться пояснениями (что считается, по каким данным),
 - при необходимости упоминать допущения и ограничения.

## 4. Ограничения и практические моменты
- Время ответа на один вопрос не должно быть чрезмерным
- Количество обращений к call_llm в рамках ответа нужно держать разумным.
- Кэширование результатов LLM-вызовов
